Workflow

How the project should grow

Start small. Add files, create a pull request, ask Codex to review it, then improve the project step by step. This prevents the usual software tragedy: ten folders, zero working app.

  1. Create filesAdd website or app files to the repository.
  2. Open Pull RequestDescribe what changed clearly.
  3. Tag CodexAsk for review, bug fixes, or improvements.
  4. Merge safelyReview changes before merging to main.
